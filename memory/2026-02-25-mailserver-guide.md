## DNS Records Required

Before starting, configure these DNS records:

### Required:

1. **A Record**
   - Type: A
   - Name: `mail`
   - Value: `217.76.55.124`
   - TTL: 3600

2. **MX Record**
   - Type: MX
   - Name: `@`
   - Value: `mail.biodockify.com`
   - Priority: 10

### Recommended (for email deliverability):

3. **SPF Record**
   - Type: TXT
   - Name: `@`
   - Value: `v=spf1 ip4:217.76.55.124 -all`

4. **DMARC Record**
   - Type: TXT
   - Name: `_dmarc`
   - Value: `v=DMARC1; p=none; rua=mailto:dmarc@biodockify.com`

### For DKIM (after mail server starts):

5. **DKIM Record**
   - Generated by docker-mailserver
   - Location: `$MAIL_DIR/config/opendkim/keys`
   - Add the public key as TXT record

## Quick Start

### Option 1: Automated Setup Script

```bash
cd ~/mailserver
./setup.sh
```

This script will:
1. Check if DNS is configured
2. Prompt for email account details
3. Create first email user
4. Start the mail server

### Option 2: Manual Setup

**Step 1: Create Email User**

```bash
cd ~/mailserver
mkdir -p config/postfix/accounts

# Create user (username:password format)
echo "admin@biodockify.com:yourpassword" > config/postfix/accounts
```

**Step 2: Start Mail Server**

```bash
docker compose up -d
```

**Step 3: Check Status**

```bash
docker ps
docker logs mailserver
```

### Add New Email User

```bash
cd ~/mailserver
echo "newuser@biodockify.com:password" >> config/postfix/accounts
docker exec mailserver setup email add newuser@biodockify.com password
```

### Remove Email User

```bash
docker exec mailserver setup email del user@biodockify.com
```

### List All Users

```bash
docker exec mailserver setup email list
```

### Update Postfix Maps

After adding/removing users:

```bash
docker exec mailserver postfix reload
```

### Restart Mail Server

```bash
cd ~/mailserver
docker compose restart
```

## Security

### Fail2Ban

Enabled by default. Bans IPs after multiple failed login attempts.

**Check bans:**
```bash
docker exec mailserver fail2ban-client status
docker exec mailserver fail2ban-client status postfix
```

**Unban IP:**
```bash
docker exec mailserver fail2ban-client set postfix unbanip 1.2.3.4
```

### SpamAssassin

Enabled by default. Filters spam emails.

**Check spam score:**
```bash
docker exec mailserver spamassassin --lint
```

### Firewall

Ensure these ports are open:
```bash
sudo ufw allow 25/tcp
sudo ufw allow 587/tcp
sudo ufw allow 465/tcp
sudo ufw allow 993/tcp
sudo ufw reload
```

## Troubleshooting

### Container Won't Start

```bash
docker logs mailserver
cd ~/mailserver
docker compose up -d --force-recreate
```

### Can't Send Email

1. Check port 25 is not blocked:
   ```bash
   nc -zv mail.biodockify.com 25
   ```

2. Check logs for errors:
   ```bash
   docker logs mailserver | grep -i error
   ```

3. Verify DNS MX record:
   ```bash
   dig MX biodockify.com
   ```

### Emails Going to Spam

1. Check SPF record is correct
2. Set up DKIM (see below)
3. Set up DMARC
4. Wait 24-48 hours for DNS propagation

### Certificate Errors

**Manual SSL setup:**

1. Generate self-signed cert:
   ```bash
   openssl req -new -x509 -days 365 -nodes \
     -out ~/mailserver/config/ssl/cert.pem \
     -keyout ~/mailserver/config/ssl/key.pem
   ```

2. Update `.env`:
   ```bash
   SSL_TYPE=manual
   SSL_CERT_PATH=/etc/ssl/mail/cert.pem
   SSL_KEY_PATH=/etc/ssl/mail/key.pem
   ```

3. Copy certs to container:
   ```bash
   docker compose down
   mkdir -p ~/mailserver/config/ssl
   docker compose up -d
   ```

## DKIM Setup (Important for Deliverability)

### Generate DKIM Key

```bash
cd ~/mailserver
docker exec mailserver setup config dkim keysize 2048 selector default
```

### Get DKIM Record

```bash
cat config/opendkim/keys/biodockify.com/default.txt
```

### Add to DNS

Copy the output and add as TXT record:
- Name: `default._domainkey`
- Value: (the entire DKIM record)

### Verify

After DNS propagation:
```bash
dig TXT default._domainkey.biodockify.com
```
